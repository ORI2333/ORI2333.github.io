新流程测测试方法
<!-- more -->

正常日常流程就按这个来：

1. 打开 GUI

```
cd E:\blog\my-blog
npm run blog
```
2. 写新文章
点 **新建草稿**，输入标题。它会在 Obsidian 里创建：
```
D:\Program Files\Obsidian\Obsidian_store_GM12\Blog\Posts
```
新文章默认有：
```
draft: true
```
这表示草稿，不会被同步发布。
3. 在 Obsidian 里写文章
正文写好后，如果准备发布，把 front matter 改成：
```
draft: false
```
或者删掉 `draft` 这一行。
4. 同步到博客仓库
回到 GUI，点：
```
同步到 Hexo
```
它会把 Obsidian 里的正式文章复制到：
```
E:\blog\my-blog\source\_posts
```
`draft: true` 的文章会自动跳过。
5. 本地检查点：
```
构建检查
```
如果没报错，说明 Hexo 能正常生成。
想本地看页面就点：
```
本地预览
```
然后浏览器访问：
```
http://localhost:4000/
```
6. 发布
确认没问题后点：
```
发布
```
它会执行 `npm run push`，提交并推送 `source` 分支。GitHub Actions 会自动构建并部署到 `main` 分支。
更省事的方式是：
```
一键完成
```
它会依次执行：
```
同步到 Hexo -> 构建检查 -> 发布
```
